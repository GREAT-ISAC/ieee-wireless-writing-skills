# Reviewer Response Reference

Use this for IEEE reviewer responses and revision plans.

## Response Template

```text
Comment R1.2:
[Preserve reviewer comment.]

Response:
Thank you for pointing this out. We have [specific action]. In the revised manuscript, [location placeholder] now [specific change]. The added experiment/analysis shows [bounded evidence]. 
```

If exact line numbers are unavailable:

`Location: Sec. X, paragraph Y, or AUTHOR_INPUT_NEEDED for line numbers.`

## Difficult Cases

- Missing experiment: do not claim it was done. Propose an action or ask the author.
- Reviewer asks for impossible comparison: explain availability/compatibility issue and offer closest fair alternative.
- Reviewer challenges novelty: map the difference to assumptions, objective, method, and evidence.
- Conflicting reviewers: satisfy both when possible; otherwise explain the tradeoff in the response summary.

## QA Checklist

- Every comment has an ID and response.
- Every promised change has a manuscript location.
- Every new experiment has a figure/table reference or placeholder.
- Tone is respectful and technical.
- No fabricated line numbers, citations, or results.
